# Ultra96-v2 AI칩 자율주행 시스템 요약

이 문서는 현재 저장소에 있는 주행 코드와 노트북을 바탕으로, 차량 구동 흐름, 주행 알고리즘, 테스트 및 개선 내용을 정리한 문서다.

## 1. 프로젝트 개요

- 대상 플랫폼: Xilinx Ultra96-v2 + PYNQ + DPU
- 주요 목표: 카메라 입력으로 차선을 인식하고, 차량을 안정적으로 주행시키는 자율주행 시스템 구현
- 핵심 방식:
  - 카메라 프레임을 BEV(Bird's-Eye-View)로 변환
  - DPU에서 YOLOv3-tiny 기반 차선 검출 수행
  - 검출된 차선 중심값을 바탕으로 조향 목표값 계산
  - 포텐셔미터 피드백을 이용한 폐루프 제어로 실제 조향 모터 제어

---

## 2. 차량 구동 방식

### 2.1 카메라 입력 및 전처리

1. 카메라 입력
   - OpenCV의 VideoCapture를 통해 카메라 프레임을 수집한다.
   - 기본 입력 해상도는 640x480이며, 영상은 이후 처리용으로 변환된다.

2. BEV 전처리
   - 입력 영상에 Perspective Transform을 적용해 Bird's-Eye-View 형태로 변환한다.
   - 차선이 차량 기준으로 정렬된 형태로 보이도록 만들어, 학습/추론 모두 동일한 전처리 파이프라인을 사용한다.

3. ROI 추출 및 리사이즈
   - BEV 영상에서 하단 영역만 잘라낸 뒤, 256x256 크기로 리사이즈한다.
   - 이 단계는 DPU 입력 형식과 모델 입력 크기에 맞추기 위한 과정이다.

### 2.2 DPU 사용 여부

- 네, DPU를 사용한다.
- YOLOv3-tiny 모델을 DPU에서 실행하여 차선 박스를 검출한다.
- 모델 파일은 보통 tiny-yolov3_256.xmodel 형태로 사용되며, 256x256 입력에 맞춰 컴파일된 모델을 로드한다.

### 2.3 모터 제어 방식

- 차량 구동은 두 종류의 모터로 나뉜다.
  - 구동 모터: 전진/후진 속도 제어
  - 조향 모터: 좌/우 회전 제어

- 구동 모터 제어는 왼쪽/오른쪽 바퀴에 각각 PWM 기반 duty 값을 부여하는 방식이다.
- 조향 모터는 포텐셔미터 ADC값을 읽어 실제 핸들 위치와 목표값을 비교하는 폐루프 방식으로 제어된다.

### 2.4 조향 제어 방식

- 조향 목표값은 카메라 기반 비전 결과로 계산된다.
- 실측 값은 조향 포텐셔미터 ADC를 통해 읽는다.
- 목표값과 실제값의 차이를 비교해 좌회전, 우회전, 직진 중 하나를 선택한다.
- 조향 제어는 빠른 주기로 반복되며, 일반적으로 100Hz 수준으로 동작한다.

### 2.5 전체 구동 흐름

카메라 입력 → BEV 변환 → ROI 추출 → 256x256 리사이즈 → DPU 추론 → 차선 중심 검출 → 조향 목표값 계산 → 조향/구동 모터 제어

---

## 3. 주행 알고리즘 설명

### 3.1 사용된 AI 모델

- 사용 모델은 YOLOv3-tiny 기반의 경량 객체 검출 모델이다.
- DPU에서 실행 가능하도록 양자화/컴파일된 모델을 사용한다.
- 주행 중에는 차선 영역을 박스로 검출하여, 차선 위치를 추정한다.

### 3.2 차선 및 중심선 인식 방법

1. 이미지 전처리 후 DPU 추론
   - BEV 영상과 ROI를 거친 이미지를 모델에 입력한다.

2. 박스 검출
   - 모델은 차선 후보 박스를 출력한다.
   - 이 중 차량 우측 차선에 해당하는 박스를 선택한다.

3. 중심 좌표 계산
   - 선택된 박스의 중심 x좌표를 계산한다.
   - 이 중심값이 차량이 따라가야 할 기준 위치로 사용된다.

### 3.3 조향값 계산 방식

- 차선 중심값이 기준값과 얼마나 차이나는지를 계산한다.
- 기준값을 기준으로 오프셋을 구하고, 이를 조향 목표값으로 변환한다.
- 구현에서는 다음과 같은 개념이 사용된다.
  - 중심이 오른쪽으로 이동하면 우회전 또는 좌회전 중 하나로 해석
  - 중심이 왼쪽으로 이동하면 반대로 조향
  - 조향값은 제한 범위 안에서 클리핑되어 안정적으로 동작하도록 한다.

- 노트북 구현에서는 다음과 같은 방식이 사용된다.
  - 차선 중심값과 기준 좌표의 차이를 이용해 조향 타겟을 계산
  - 실제로는 선형 비례 방식과 함께, 시각적 각도 기반 계산도 함께 사용된다.

### 3.4 속도 제어 방식

- 속도는 기본적으로 고정값으로 설정되어 동작한다.
- 구동 모터는 동일한 속도 값을 왼쪽/오른쪽에 적용해 전진하도록 구성된다.
- 테스트 중에는 속도 값을 낮게 시작해 안정성을 확인한 뒤, 필요 시 조정한다.

### 3.5 사용한 제어 알고리즘

- 조향 제어는 폐루프 제어 기반이다.
- 비전 결과로 목표 조향값을 만들고, 실제 핸들 위치는 ADC로 측정한다.
- 그 차이를 이용해 좌/우/직진 명령을 결정한다.
- 구현상으로는 다음 요소가 포함된다.
  - 히스테리시스 구간: 목표값과 실제값 차이가 작으면 정지 명령 유지
  - 방향 판단: 실제값이 목표보다 크면 좌측 보정, 작으면 우측 보정
  - 트림 보정: 실제 직진 상태와 센서 기준값의 오차를 보정

즉, 전체 제어 방식은 “비전 기반 목표값 생성 + 센서 기반 폐루프 보정” 구조다.

## 4. 테스트 및 개선 내용

### 4.1 초기 테스트에서 확인한 사항

- 조향 방향이 실제 차량 배선과 반대로 동작할 가능성이 있어, 바퀴를 지면에서 띄운 상태에서 먼저 확인하는 절차가 필요했다.
- ADC 값과 실제 핸들 위치의 관계를 먼저 확인해야 하므로, 캘리브레이션 단계가 필수적이었다.
- 초기에는 조향이 과도하게 민감하거나, 차선이 잘못 인식될 경우 차량이 크게 휘갈릴 수 있었다.

### 4.2 주요 개선 내용

1. 조향 민감도 조절
   - STEER_GAIN 값을 조정해 차선 중심 오프셋에 대한 조향 반응 강도를 조절했다.
   - 너무 민감하면 진동이 심해지고, 너무 둔하면 곡선에서 잘 따라가지 못했다.

2. 조향 중립 보정
   - STEER_TRIM 값을 이용해 실제 직진 상태와 센서 기준값의 편차를 보정했다.
   - 이 보정 덕분에 차량이 한쪽으로 쏠리는 현상을 줄일 수 있었다.

3. 조향 방향 보정
   - 차량의 실제 회전 방향과 모델이 해석한 방향이 다를 수 있어, STEER_DIR를 조정했다.
   - 이는 실제 주행 환경에서 가장 자주 수정하는 파라미터 중 하나다.

4. 기준 위치 조정
   - REF_X 값을 조정해, 차선 중심이 기준 위치에서 어떻게 해석되는지 튜닝했다.
   - 차선이 화면에서 어느 위치에 있을 때 조향 목표가 생성되는지 조정하는 과정이다.

5. 속도 조정
   - 초기에는 안전을 위해 낮은 속도로 테스트했고, 이후 주행 안정성을 고려해 속도 값을 조정했다.
   - 속도가 너무 높으면 인식 지연과 제어 지연이 겹쳐 차량이 흔들릴 수 있다.

6. 인식 오류 개선
   - BEV 변환, ROI 자르기, 256x256 리사이즈를 통해 입력 이미지의 품질을 안정화했다.
   - 차량이 차선 검출을 놓치거나 잘못된 박스를 선택할 때를 대비해, 검출 기준과 임계값을 조정했다.

7. 디버깅 및 분석 강화
   - 블랙박스 기록 기능을 통해 프레임 단위로 인식/판단/제어 정보를 저장했다.
   - 이를 통해 시각 인식 문제인지, 조향 목표 계산 문제인지, 모터 제어 문제인지 분리해 분석할 수 있었다.

### 4.3 테스트 결과 정리

- 가장 중요한 개선 포인트는 “인식 안정성 + 조향 보정 + ADC 캘리브레이션” 이었다.
- DPU 기반 차선 인식이 가능했지만, 실제 주행에서는 조향의 미세한 오차가 차량의 안정성에 큰 영향을 준다.
- 따라서 비전 모델 출력만이 아니라, 센서 기반 폐루프 제어와 파라미터 튜닝이 함께 필요하다.

---

## 5. 한 줄 요약

이 프로젝트는 카메라 입력으로 차선을 인식하고, DPU 기반 YOLOv3-tiny 모델로 차선 중심을 추정한 뒤, 조향 목표값을 계산하고 포텐셔미터 피드백 기반 폐루프 제어로 실제 차량을 주행시키는 구조다. 핵심은 “인식의 정확도”와 “조향 제어의 안정성”을 동시에 맞추는 것이다.
